1.0.26 - 2019-1-15

Changed

  • Photo #1808 - merge branch ‘DSphoto-4.13.4’ to master
    • 該branch主要修正PhotoBackup相關bug以及refactor, 如果沒有用到PhotoBackup的project並不會受影響
    • 以下列舉使用到PhotoBackup的project需要注意的更動:
      1. 把 SYNOPhotoLibraryHandler 的初始函式，拿掉 isSkipAll 參數。並在 SYNOPhotoBackupAssetHandlerProtocol 中增加 isSkipAll 與 setSkipAll: 介面
        • 原因
          • 讓 skipAll 可以被各 project 儲存在 UserDefaultSetting 中，app 重啟後仍可以讀到設定，避免初次掃描相片未完時，被重啟後，失去了 skipAll 設定，變成全部上傳。
        • 注意事項
          • 各 project 要實作 isSkipAll 與 setSkipAll: 介面
          • 如果有使用到SYNOPhotoLibraryHandler 的初始函式需要對應修改解決compile error
      2. 以 BGUploadErrorServiceDisabled 取代 BGUploadErrorAPINotExist
        • 原因
          • 原本 APINotExist 是在 Moments 上的 error code，但 Photo 並沒有，因此以較中性的 ServiceDisabled 取代。
        • 注意事項
          • 有使用到BGUploadErrorAPINotExist的地方以BGUploadErrorServiceDisabled取代
      3. 增加 BGUPloadErrorSessionTimeout, BGUploadErrorAssetSize 等 error code
        • 新增加的Error code對project沒有影響，可視需要使用
